# Michael Nygard 的決策記錄範本

這是出自 [Documenting architecture decisions - Michael Nygard](http://thinkrelevance.com/blog/2011/11/15/documenting-architecture-decisions) 一文的範本。
你可以使用 [adr-tools](https://github.com/npryce/adr-tools) 來管理 ADR 檔案。

在每個 ADR 檔案中，撰寫以下章節：

# 標題

## 狀態

目前的狀態為何？例如：提議中（proposed）、已接受（accepted）、已拒絕（rejected）、已棄用（deprecated）、已取代（superseded）等。

## 背景脈絡

是什麼議題促使我們做出這個決策或變更？

## 決策

我們提議和/或正在執行的變更是什麼？

## 結果

因為這項變更，什麼事情變得更容易或更困難？
