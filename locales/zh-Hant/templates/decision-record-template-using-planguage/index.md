# 使用 Planguage 的決策記錄範本

參見 https://www.iaria.org/conferences2012/filesICCGI12/Tutorial%20Specifying%20Effective%20Non-func.pdf

## 什麼是 Planguage？

Planguage 是一種規劃語言（Planning Language），使用以下關鍵字：

* Tag（標籤）：唯一且持久的識別碼
* Gist（要旨）：需求或涵蓋領域的簡要摘要
* Requirement（需求）：詳述需求本身的文字
* Rationale（理由）：證明需求合理性的推理
* Priority（優先級）：優先級聲明及對資源的需求
* Stakeholders（利害關係人）：受需求實質影響的相關方
* Status（狀態）：需求的狀態（草稿、已審查、已承諾等）
* Owner（負責人）：負責實作需求的人員
* Author（作者）：撰寫需求的人員
* Revision（修訂版）：陳述的版本號
* Date（日期）：最近一次修訂的日期
* Assumptions（假設）：如果現在或之後不成立可能導致問題的任何事項
* Risks（風險）：可能導致故障、延遲或其他負面影響的任何事項
* Defined（定義）：術語的定義（最好使用詞彙表）
