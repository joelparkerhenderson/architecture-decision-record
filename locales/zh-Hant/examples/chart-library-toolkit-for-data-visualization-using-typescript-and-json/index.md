# 架構決策記錄：使用 TypeScript 和 JSON 進行資料視覺化的圖表函式庫工具組

**主要目標：**
選擇一個進階圖表工具組，用於建立互動式視覺化，專注於使用 TypeScript 和 JSON 呈現金融資料、科學資料和政府資料。該函式庫應提供強大的功能、靈活性，並且是開源的。

### 背景與需求：

1. **敏捷開發（高優先級）**：作為一家新創公司，快速迭代、原型開發和開發靈活性至關重要。圖表函式庫必須支援快速的開發週期。

2. **圖表類型（高優先級）**：
   - **甜甜圈圖（Doughnut Chart）**
   - **雷達圖（Radar Chart）**
   - **聚類流程圖（Clustering Process Chart）**
   - **帶時間軸的面積圖（Area Chart with Time Axis）**
   - **K 線圖（Candlestick Chart）**
   - **南丁格爾玫瑰圖（Nightingale Chart）**
   - **地理 SVG 地圖（Geo SVG Map）**

   這些圖表類型對於視覺化複雜資料集特別重要，例如金融趨勢、科學指標和地理資訊。

3. **免費且開源（高優先級）**：工具組應為開源，以避免授權費用、提供透明度，並提供客製化的靈活性。

4. **低優先級標準**：
   - **執行速度**：雖然效能很重要，但並非此決策的首要考量。
   - **可擴展性**：雖然可擴展性通常很重要，但當前的需求是建構一個可隨時間成長的 MVP（最小可行產品）。可擴展性問題可以稍後解決。
   - **向後相容性**：對初始建構來說並非首要考量，只要函式庫是現代的且有積極維護即可。

### 評估的函式庫：

1. **Apache ECharts**
2. **Chart.js**
3. **ApexCharts**
4. **AG Charts**
5. **Highcharts**
6. **Carbon Charts**
7. **Layer Cake**
8. **D3.js**

---

### 1. **Apache ECharts**

**概述**：
Apache ECharts 是一個強大、靈活的互動式可自訂視覺化圖表函式庫。它支援多種圖表類型，在複雜的動態視覺化方面特別強大。

**優勢**：
- **進階互動性**：ECharts 在提供互動式圖表方面表現出色，具備縮放、平移和動態資料更新等功能。
- **甜甜圈圖、雷達圖、K 線圖、地理 SVG 地圖**：ECharts 支援許多所需的圖表類型，包括甜甜圈圖、雷達圖、K 線圖和地理地圖視覺化。
- **免費且開源**：ECharts 是開源函式庫，符合新創公司注重成本的特性，並提供修改程式碼的自由。
- **靈活性與可擴充性**：高度可自訂，廣泛支援動畫、自訂視覺化和進階圖表技術。

**劣勢**：
- **學習曲線**：ECharts 雖然強大，但由於其靈活性和豐富的 API，學習曲線可能較為陡峭。
- **文件複雜度**：文件雖然全面，但對剛入門的開發者可能會感到難以消化。

**結論**：
ECharts 非常適合本專案，因為它支援互動式圖表，包括所有所需的類型如 K 線圖、雷達圖和地理地圖。其開源特性符合專案對靈活性和成本效益的需求。

---

### 2. **Chart.js**

**概述**：
Chart.js 是一個簡單、易用的圖表函式庫，用於建構常見的圖表類型。它以簡潔和易於整合著稱。

**優勢**：
- **易用性**：Chart.js 非常容易設定和使用，學習曲線極低。
- **開源**：Chart.js 是免費且開源的，對降低成本至關重要。
- **常見圖表類型**：它支援基本圖表如甜甜圈圖、面積圖、雷達圖和折線圖，涵蓋了大部分基本需求。

**劣勢**：
- **進階圖表受限**：Chart.js 原生不支援複雜的圖表類型如 K 線圖、地理 SVG 地圖或聚類流程圖。雖然可以透過外掛或自訂來新增這些功能，但不如其他函式庫那麼簡單。
- **互動性**：雖然 Chart.js 支援基本互動（如工具提示和懸停效果），但互動功能不如 ECharts 或 D3.js 進階。

**結論**：
Chart.js 適合簡單、快速的專案，但由於缺乏對複雜圖表類型的支援，不適合具有 K 線圖和地理地圖等進階需求的資料密集型應用程式。它適合原型開發，但對於所需的圖表類型，建議使用更進階的工具。

---

### 3. **ApexCharts**

**概述**：
ApexCharts 是一個現代圖表函式庫，提供多種圖表類型，專注於透過易用的 API 實現互動式視覺化。

**優勢**：
- **互動功能**：ApexCharts 提供具有工具提示、縮放、平移和更新功能的互動式圖表。
- **支援金融和科學圖表**：它支援多種圖表類型，包括 K 線圖、雷達圖和面積圖。
- **易用性**：它擁有直觀的 API，易於整合到專案中。
- **免費且開源**：ApexCharts 提供適合多種使用情境的免費開源版本。

**劣勢**：
- **複雜自訂受限**：雖然它提供許多功能，但對於高度複雜或自訂的圖表需求，自訂選項不如 ECharts 或 D3.js 靈活。
- **地理地圖**：ApexCharts 原生不支援地理地圖或聚類流程圖，而這些是本專案所需的。

**結論**：
ApexCharts 因其易用性和互動性而成為強力候選，但在某些進階圖表類型上有所不足，特別是對地理地圖和聚類圖的需求。它適合較簡單的圖表，但缺少部分所需功能。

---

### 4. **AG Charts**

**概述**：
AG Charts 是一個商業級圖表函式庫，專為效能和精確度而設計。它非常適合建立金融、科學和商業儀表板。

**優勢**：
- **進階圖表類型**：AG Charts 支援許多進階圖表類型，包括 K 線圖、面積圖、雷達圖等。它還與其他 AG-Grid 產品深度整合。
- **高效能**：在處理大型資料集時提供出色的效能。
- **互動性**：AG Charts 支援多種互動功能，如縮放、工具提示和動態更新。

**劣勢**：
- **非完全免費**：雖然 AG Charts 提供免費版本，但完整功能版本是付費的，這可能成為希望降低成本的新創公司的障礙。
- **複雜性**：雖然功能豐富，但對較簡單的專案可能過於複雜，與其他選項相比需要更多的設定和配置。

**結論**：
AG Charts 功能強大且豐富，但由於其商業性質和成本結構，可能不是最佳選擇。其適用性取決於預算是否能承擔付費版本，或是否偏好開源替代方案。

---

### 5. **Highcharts**

**概述**：
Highcharts 是一個受歡迎的圖表函式庫，以其廣泛的圖表類型和強大的自訂選項聞名。

**優勢**：
- **全面的圖表類型**：Highcharts 支援多種圖表，包括 K 線圖、雷達圖、面積圖和地理地圖。
- **互動且動態**：Highcharts 提供豐富的互動功能，包括下鑽（Drill-down）、縮放和平移。
- **易用性**：它擁有友善的 API 和良好的文件，容易上手。

**劣勢**：
- **商業授權**：雖然 Highcharts 為非商業用途提供免費版本，但商業授權費用昂貴，這對新創公司可能是重大缺點。
- **學習曲線**：雖然不如 ECharts 陡峭，但 Highcharts 的學習曲線對初學者來說仍具挑戰性。

**結論**：
Highcharts 功能豐富，但其商業授權使其較不適合開源、注重成本的專案。全面的圖表選項是優勢，但授權問題限制了其在此使用情境中的吸引力。

---

### 6. **Carbon Charts**

**概述**：
Carbon Charts 是由 IBM 開發的圖表函式庫，專為建立視覺美觀且高度可自訂的圖表而設計。

**優勢**：
- **可自訂性**：Carbon Charts 允許對圖表外觀和行為進行廣泛自訂。
- **開源**：它是免費且開源的，符合專案對經濟實惠解決方案的需求。
- **支援常見圖表**：它支援常見圖表類型如甜甜圈圖、雷達圖和面積圖，但缺乏對地理地圖或 K 線圖等更進階類型的支援。

**劣勢**：
- **進階圖表類型受限**：它不支援地理地圖、聚類流程圖或 K 線圖，而這些對專案至關重要。
- **較小的生態系**：與 ECharts 或 Highcharts 等較大的圖表函式庫相比，Carbon Charts 的社群和生態系較小。

**結論**：
Carbon Charts 是開源且可自訂的，但缺乏本專案所需的更複雜圖表類型的支援。它更適合較簡單的圖表需求。

---

### 7. **Layer Cake**

**概述**：
Layer Cake 是一個資料視覺化函式庫，專為建立靈活的分層視覺化而設計。

**優勢**：
- **可自訂的分層**：它為複雜視覺化提供強大的分層選項。
- **開源**：它是免費且開源的，是注重預算的專案的可行選項。

**劣勢**：
- **文件有限**：Layer Cake 缺乏豐富的文件和社群支援，與更成熟的函式庫相比較難使用。
- **非專為圖表設計**：Layer Cake 更適合非圖表視覺化，因此其開箱即用的圖表選項有限。

**結論**：
雖然對獨特的視覺化有其趣味，但 Layer Cake 不適合 K 線圖或雷達圖等傳統圖表需求。它更適合標準圖表範圍之外的自訂視覺化。

---

### 8. **D3.js**

**概述**：
D3.js 是一個強大的 JavaScript 函式庫，透過 HTML、SVG 和 CSS 建立資料驅動的視覺化。

**優勢**：
- **無與倫比的靈活性**：D3.js 幾乎可以建立任何類型的自訂視覺化，使其在進階互動式圖表方面極為強大。
- **豐富的功能**：它支援所有所需的圖表類型，包括地理地圖、聚類圖等。
- **可自訂性**：D3.js 的自訂程度無可比擬，允許開發者建構高度客製化的視覺化。

**劣勢**：
- **陡峭的學習曲線**：D3.js 的學習曲線陡峭，與其他函式庫相比整合更為複雜。
- **耗時**：在 D3.js 中建構圖表可能非常耗時，尤其是對於 K 線圖或甜甜圈圖等常見圖表。

**結論**：
D3.js 在進階、自訂圖表方面極為強大，但由於其陡峭的學習曲線和開發時間，對許多典型使用情境來說過於複雜。它最適合其他圖表函式庫無法提供所需自訂程度的情況。

---

### 結論

根據專案需求評估各函式庫後，**Apache ECharts** 脫穎而出成為最佳選擇。它支援完整的所需圖表類型，包括地理地圖、K 線圖和聚類圖。它是開源的、功能豐富且高度互動，完全符合專案的目標。雖然 **D3.js** 提供最大的靈活性，但其複雜性和時間投入使其對希望快速迭代的新創公司來說不太理想。**ApexCharts** 和 **Chart.js** 是較簡單專案的良好替代方案，但缺乏對進階圖表類型的支援。
