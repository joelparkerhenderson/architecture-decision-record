# 架構決策記錄：程式碼編輯器（Programming Code Editors）

## 背景

程式碼編輯器是開發者撰寫和編輯程式碼的必備工具。市面上有許多程式碼編輯器，各自擁有不同的功能、優勢和劣勢。本 ADR 的目的是記錄針對程式碼編輯器所做的架構決策。

## 優先事項

程式碼編輯器的架構應優先考慮以下幾點：

* **模組化（Modularity）**：程式碼編輯器應以模組化方式設計，允許開發者根據需要進行自訂和擴展。這使得架構具有彈性，能夠適應不同開發者和團隊的需求。

* **效能（Performance）**：程式碼編輯器應具備高效能和即時回應能力，讓開發者能夠高效工作，不會被使用的工具拖慢速度。

* **使用者介面（User Interface）**：使用者介面應直覺且易於使用，讓開發者能專注於程式碼本身，而不是與編輯器搏鬥。

* **可擴展性（Extensibility）**：程式碼編輯器應設計為易於透過第三方外掛程式和整合進行擴展。

* **相容性（Compatibility）**：程式碼編輯器應與廣泛的程式語言和技術相容，使其成為對各類開發者都實用的工具。

## 決策

基於這些優先事項，程式碼編輯器的架構應包含以下元件：

* **核心（Core）**：此元件提供程式碼編輯器的基本功能，例如語法高亮、文字編輯和檔案管理。

* **UI**：此元件提供程式碼編輯器的使用者介面，包括選單、工具列和鍵盤快捷鍵。

* **外掛程式（Plugins）**：此元件允許開發者透過安裝第三方外掛程式來擴展程式碼編輯器的功能。外掛程式可以提供額外功能，例如程式碼自動完成、程式碼檢查（linting）或除錯。

* **整合（Integrations）**：此元件允許程式碼編輯器與其他工具和技術整合，例如版本控制系統、建置系統或除錯工具。

## 理由

程式碼編輯器的模組化允許開發者根據需要進行自訂和擴展。這很重要，因為不同的開發者和團隊有不同的需求和工作流程，而彈性的架構能夠容納這些差異。

* **效能**：至關重要，因為開發者需要能夠高效工作，不被工具拖慢。高效能的程式碼編輯器對生產力至關重要，並能幫助開發者維持專注和注意力。

* **UI**：很重要，因為它讓開發者能專注於程式碼而非與編輯器搏鬥。這可以帶來更好的生產力，減少開發者的挫折感。

* **可擴展性**：強大之處在於它讓程式碼編輯器能適應不同的需求和工作流程。第三方外掛程式和整合可以提供核心編輯器未包含的額外功能和能力。

* **相容性**：有價值之處在於它讓程式碼編輯器能與廣泛的程式語言和技術一起使用。這使得編輯器成為對各類開發者更實用的工具。

核心、外掛程式、整合和 UI 元件提供了清晰的關注點分離（separation of concerns），並允許模組化架構的輕鬆擴展和自訂。此架構具有彈性、高效能，並與廣泛的程式語言和技術相容，使其成為對開發者實用的工具。
