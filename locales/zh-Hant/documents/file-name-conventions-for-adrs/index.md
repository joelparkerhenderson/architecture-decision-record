# 檔案命名慣例

如果你選擇使用一般文字檔來建立 ADR，那麼你可能會想制定自己的 ADR 檔案命名慣例。

我們偏好使用具有特定格式的檔案命名慣例。

範例：

  * choose-database.md

  * format-timestamps.md

  * manage-passwords.md

  * handle-exceptions.md

我們的檔案命名慣例：

  * 名稱使用現在式祈使語氣的動詞片語。這有助於提升可讀性，並與我們的提交訊息（commit message）格式一致。

  * 名稱使用小寫字母和連字號（與本儲存庫相同）。這是在可讀性與系統可用性之間取得的平衡。

  * 副檔名為 Markdown。這對於輕鬆格式化很有幫助。
